General Execution Steps - Parallel System

In a parallel pipeline system, the execution of instructions is divided into stages, and multiple stages can be active at the same time, allowing for the concurrent execution of multiple instructions. The stages typically include:

  1. Instruction Fetch (IF): This stage involves fetching the instruction from memory.

  2. Instruction Decode (ID): In this stage, the instruction is decoded to determine the operation to be performed and the operands involved.

  3. Execute (EX): This stage is where the actual operation specified by the instruction is carried out. For example, addition, subtraction, etc.

  4. Memory Access (MEM): This stage is responsible for accessing memory if needed (e.g., for load or store operations).

  5. Write Back (WB): In this stage, the result of the instruction is written back to the appropriate register.

Data Hazard

Consider the below two instructions

  • ADD R1 R2 R3
  • SUB R4 R1 R3

Let’s break down the execution of the instructions ADD R1 R2 R3 and SUB R4 R1 R3 in a parallel pipeline system:

Instruction 1: ADD R1 R2 R3

  1. Clock Cycle 1 (CC1):

    • IF: Fetch the instruction ADD R1 R2 R3.
    • ID: Decode the instruction to determine it’s an addition operation and that it involves registers R1, R2, and R3.

  2. Clock Cycle 2 (CC2):

    • EX: Perform the addition operation (R1 = R2 + R3).

  3. Clock Cycle 3 (CC3):

    • MEM: This stage may be idle as there’s no memory access needed for this instruction.

  4. Clock Cycle 4 (CC4):

    • WB: Write the result of the addition back to register R1.

Instruction 2: SUB R4 R1 R3

Since there’s a data dependency (RAW hazard) on register R1, the execution of this instruction will be affected by the result of the first instruction.

  1. Clock Cycle 1 (CC1):

    • IF: Fetch the instruction SUB R4 R1 R3.
    • ID: Decode the instruction to determine it’s a subtraction operation and that it involves registers R4, R1, and R3.

  2. Clock Cycle 2 (CC2):

    • EX: This stage may be stalled because the value of R1 (which is produced by the first instruction) is not yet available. The processor may need to insert a bubble or idle cycle here.

  3. Clock Cycle 3 (CC3):

    • MEM: This stage may be idle as there’s no memory access needed for this instruction.

  4. Clock Cycle 4 (CC4):

    • WB: This stage may also be stalled because the result of the subtraction operation cannot be written back until the execution stage is complete.

In this scenario, the pipeline will experience a stall in clock cycles 2 and 4 due to the data hazard. The processor needs to handle this by either inserting idle cycles or using techniques like forwarding to resolve the data hazard.

Keep in mind that in an actual processor, there might be additional stages or optimizations, and the exact behavior could vary depending on the specific architecture and design choices.

Fix 1: Forwarding

Data hazard can be prevented in the given case using the forwarding (also known as data hazard forwarding or data forwarding) technique.

Forwarding involves directly passing the result of a previous instruction to a subsequent instruction that requires it, bypassing the need to write it to memory first. This helps to eliminate stalls in the pipeline due to data dependencies.

To prevent the data hazard using forwarding, the pipeline stages would operate as follows:

Instruction 2: SUB R4 R1 R3

Since there’s a data dependency on register R1, the forwarding technique allows us to pass the result from the ADD operation directly to the SUB operation, bypassing the need to write R1 to memory first.

  1. Clock Cycle 1 (CC1):

    • IF: Fetch the instruction SUB R4 R1 R3.
    • ID: Decode the instruction to determine it’s a subtraction operation and that it involves registers R4, R1, and R3.

  2. Clock Cycle 2 (CC2):

    • EX: The processor detects that the value needed for R1 is available from the previous instruction’s execution stage. It forwards this value to the SUB operation (R4 = R1 - R3).
    • In other words, the result of the addition operation is directly fetched from ALU rather than stalling until it is written back to memory

  3. Clock Cycle 3 (CC3):

    • MEM: This stage may be idle as there’s no memory access needed for this instruction.

  4. Clock Cycle 4 (CC4):

    • WB: Write the result of the subtraction back to register R4.

With the forwarding technique, there are no stalls in the pipeline, and the instructions can be executed back-to-back without waiting for data to be written to memory. This helps improve the overall throughput and performance of the processor.
